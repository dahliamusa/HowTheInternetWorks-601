DNS

DNS makes it possible for us to use domain names to search for websites instead of IP addresses

Domain Name Server (DNS) contains a large database that is used to associate domain names with IP addresses. When you look up a website, your computer uses DNS to find the IP address associated with the domain name you entered, and then uses the address to connect your computer with the website. DNS servers are networked in a distributed hierarchy and are divided into zones, as one DNS server cannot handle all the requests sent from the billions of existing devices. Using zones, DNS servers split up responsibility for major top-level domains (TLDs) like .com, .org and .net.

DNS was created to be an open and public communication protocol, and because of its openness, DNS is susceptible to cyber attacks. For example, hackers may enter the DNS server and make changes so that a domain name matches the wrong IP address. Hackers can send users to an imposter website, leading them to believe they are on one website when they are actually on another. HTTPS protocols work to keep the Internet secure to prevent such attacks from happening.
