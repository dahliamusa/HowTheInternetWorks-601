HTTP and HTTPS Protocols

You can tell if HTTPS is active ny looking for the lock icon in the address bar

Hypertext Transfer Protocol (HTTP) is a request-response protocol that is used when you go to a website in your browser. After you enter the website’s URL in your address bar and click enter, your computer connects with a Domain Name Server (DNS) to get the IP address of the associated web server. Your computer then begins communicating with the web server, which is usually thousands of miles away. Your computer will send a GET request to tell the server to retrieve the desired website. Using the data from the web server, your computer is able to access the website for you from the browser.

If you are logging into a website, your computer will send a POST request to the web server. The POST request will tell the web server to verify your information, log you in and send your computer the webpage. With the webpage, the server also sends your computer some cookie data which the browser sees and saves. The cookie is an ID number that identifies you in that website. The next time you log in, your browser will know to send the cookie data to the web server so the web server knows where the request is coming from.

The Internet is open, connections are shared and information is sent in plain text, and because of this, hackers may try to access private information over the Internet. Safe websites prevent this from happening by asking your web browser to communicate on a secure channel using secure sockets layer (SSL) and its successor, transport layer security (TLS). SSL and TLS act as a layer of security wrapped around communications to protect them from snooping or tampering. SSL or TLS are active when a lock appears in the address bar next to “https.” HTTPS ensures that the website you are accessing is safe and that your information is protected.

When a website asks your browser to engage in secure connection, it first provides a digital certificate that proves that it is the website it claims to be. Certificates are published by certificate authorities, who verify identities of websites and issue certificates for them. If a website tries to start a connection without a certificate, the browser will warn you.
